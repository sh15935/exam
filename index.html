Question 2

Explain the four types of user profiles in Active Directory and indicate how to create a Mandatory Profile.

(2 points)
âœ… Complete Correction:
The 4 types of user profiles:

1. Local Profile:
  • Created automatically at first login
  • Stored on local hard drive: C:\Users\Username
  • Does NOT follow the user to other computers
  • Each computer has its own local profile for the user

2. Roaming Profile:
  • Stored on a network server
  • Follows the user to any domain computer
  • Synchronized at login and logout
  • Path configured in AD: \\Server\Profiles$\%username%

3. Mandatory Profile:
  • Read-only version of a roaming profile
  • Changes are lost at logout
  • Ensures a standardized environment
  • Ideal for shared accounts or public kiosks

4. Temporary Profile:
  • Created when the normal profile cannot be loaded
  • All changes are lost at logout
  • Indicates a problem that needs to be resolved

Question 3

What is the difference between RODC and a classic DC? In which scenario would you use a RODC?

(1.5 points)
âœ… Complete Correction:
Classic DC (Domain Controller):
  • Database: COMPLETE and WRITABLE copy of AD database
  • Replication: Bidirectional (can write and read)
  • Authentication: Directly authenticates users
  • Modifications: Can create, modify, delete AD objects

RODC (Read-Only Domain Controller):
  • Database: READ-ONLY copy of AD database
  • Replication: Unidirectional (receives updates only)
  • Authentication: Authenticates locally but cannot modify AD
  • Modifications: Modification requests are forwarded to a classic DC
  • Password Cache: Limited and configurable password cache
  • Security: Does not store all domain passwords

ğŸ¯ RODC Usage Scenarios:
  • Remote Offices: Sites with poor physical security (no secure server room)
  • Branch Offices: Locations without qualified IT staff
  • DMZ: Demilitarized zones where security is critical
  • Public Sites: Libraries, retail stores, agencies

ğŸ’¡ Concrete Example: A bank with 50 branches. Headquarters has classic DCs. Each branch has an RODC. If an RODC is stolen from a branch, the entire domain's passwords are NOT compromised because the RODC doesn't store them all.

2.3 Explain why use OUs rather than putting all objects directly in the domain.

âœ… Correction - Benefits of OUs:
Reasons to use Organizational Units (OUs):

1. Delegation of Administration:
  • Assign specific administrators to manage only their OU (e.g., HR manager manages HR OU)
  • No need to give full domain admin rights
  • Improves security through least privilege principle

2. Group Policy Application:
  • GPOs can be linked to specific OUs
  • Different policies for different departments (IT has strict security, HR has different requirements)
  • More granular control than domain-wide policies

3. Organization and Structure:
  • Mirror your company's organizational structure
  • Easy to find and manage objects
  • Clear hierarchy and logical grouping

4. Simplified Management:
  • Bulk operations on all objects in an OU
  • Move objects between OUs as organizational structure changes
  • Filter and search objects more easily

5. Security and Permissions:
  • Apply permissions at OU level
  • Inherit permissions down the OU hierarchy
  • Control who can create, modify, or delete objects

ğŸ’¡ Example: Without OUs, a 50-employee company would have all users, computers, and groups in one flat list. With OUs, you can organize by department, making it easy to apply department-specific policies and delegate management.
